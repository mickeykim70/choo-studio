## 0. 이 프로젝트는 무엇인가 — 제1원칙: 독자는 "나"

이 사이트의 독자는 **편집장님 자신 한 명**이다. 타인이 아니다.

- **이전 시도가 무너진 이유**: 독자를 타인으로 잡으니 매번 눈치를 보게 됐다. "이게 재밌을까? 너무 어렵나?" 하는 검열관이 어깨에 올라앉으면 지친다.
- **이번엔 다르다**: 독자가 나면 그 검열관이 사라진다. 미완성·틀린 생각·낙서도 그냥 둔다. 내가 궁금하면 그게 곧 이유다.
- **모든 결정의 잣대 딱 하나**: *"이건 남 눈치인가, 내가 원하는가?"* — 남 눈치면 버린다.
- 그래서 **불필요한 것들**: SEO, 방문자 분석, 댓글, 공유 버튼, 친절 설명 강박, "발행" 부담, 꾸준함 압박, 가독성 "표준"(참고만).
- 성격은 블로그라기보다 **"내 생각 서랍 + 놀이터"**.

## 1. 세계관 — 숲속 오두막 강의동

대학 캠퍼스가 아니라 **숲**이다. 남의 건물이 아니라 **내가 숨어드는 아지트**다.

공간이 흐르면서 분위기(톤)가 자연스럽게 바뀐다 — 이게 이 사이트의 감정 동선이다:

```
숲속 오솔길    →    오두막       →    칠판          →    공책
(홈 · 발견)         (주제 · 아늑)      (글 목록 · 강의)     (글 본문 · 필기)
초록 · 산책         나무 · 따뜻        다크 · 분필          라이트 · 색연필
```

들어갈수록 밝은 숲 → 아늑한 나무 → 진지한 칠판 → 편안한 공책으로 톤이 흐른다.

---

## 2. 사이트 구조 — 공간이 곧 정보구조

| 공간 | 웹에서의 정체 | 톤 |
|---|---|---|
| **숲속 오솔길** | 홈(첫 화면) = 랜딩 | 초록·손그림 지도 |
| **오두막** | 주제 카테고리 (안경·눈 / 수학·물리 / AI / 잡담) | 나무·아늑 |
| **칠판** (오두막 안) | 그 주제의 글 목록 (index) | 다크·분필 |
| **공책** | 개별 글 본문 | 라이트·색연필 + 인쇄체 |
| **소개** | 교수(편집장님·고양이 추) 소개 | — |
| **실습실 (lab)** | ComfyUI·AI 창작물 전시 *(나중에, 작게)* | — |

핵심 정리:
- **홈페이지 ⊃ 블로그.** 사이트 전체가 "강의동(홈페이지)"이고, 글(블로그)은 그 안 오두막의 칠판에 쌓인다. 둘은 대립이 아니라 층위가 다르다.
- **오두막 클릭 → 칠판 등장 → 칠판에 글 제목들이 분필로.** 시뮬이 있는 글은 노란 분필로 표시.

---

## 3. 지름길 — 산책의 재미 + 실용 (꼭 기록)

캠퍼스형 네비게이션의 함정: 숲→오두막→칠판→글이 **4단계**라, "그 글 다시 보고 싶다" 할 때 매번 숲부터 걷는 건 귀찮다.

→ **처방**: 숲 오솔길 어딘가에 **"최근 필기 바로가기" 팻말**을 세운다.
- 산책하고 싶으면 오솔길로 천천히
- 바로 보고 싶으면 팻말로 지름길
- 재미와 실용 둘 다 챙긴다.

---

## 4. 톤 & 비주얼

- **미학**: 손그림 노트 / 칠판. 삐뚤빼뚤·색연필·분필. 차가운 3D가 아니라 **따뜻한 아날로그**.
- **톤 분담(하이브리드)**: 숲(초록 산책) · 오두막(나무 따뜻) · 칠판(다크 분필) · 공책(라이트 색연필).
- **폰트 분담**: 손글씨(제목·라벨·도해·낙서) + 인쇄체 산세리프(본문 긴 글). **섞는 게 정석** — 진짜 강의실도 인쇄 유인물에 손으로 필기한다. 어색하지 않다.
  - 한글 손글씨 후보: **Gaegu(개구)**, Nanum Pen Script / 영문: Caveat
- **화려함은 "재료"만 훔친다**: awwwards Three.js 사이트들의 부드러운 스크롤(Lenis)·스크롤 연동(GSAP)·셰이더(GLSL)·포스트프로세싱은 재료로 빌려오되, 통째로 흉내내지 않는다(그건 팀이 몇 달 만든 것 — 따라하다 지친다).
- **편집장님만의 차별점**: 손그림 톤 + **실제로 작동하는 물리 시뮬**. awwwards 60개 어디에도 없는 조합.

---

## 5. 콘텐츠 & 시뮬레이션

- **주제(오두막)**: 안경·눈 / 수학·물리 / AI(삼성SDS 출신) / 잡담.
- **시뮬 = 글의 설명 도구.** 독립 전시물이 아니라, "볼록렌즈란 무엇인가" 같은 글 **안에** 내장된다. (별도 갤러리는 오버 — 나중에 원하면 `#시뮬` 태그로 모아보기가 공짜로 됨)
- **첫 시뮬 예시**: 안구 앞 볼록렌즈의 도수↔초점 시뮬 (슬라이더로 근시/정시/원시). 오늘 회의에서 작동 데모 확인.
- **리얼리즘 사다리** (시뮬마다 목표를 따로 정한다):
  1. 교과서 작도(얇은 렌즈·점 초점) → 2. 수차 재현(구면·색수차로 초점 번짐) → 3. 정밀 광선추적(스넬 법칙) → 4. 임상 안구 모델(굴절률·조절) → 5. 3D + "환자가 보는 상"(셰이더).
  - 위로 갈수록 물리 지식·시간 비용 증가. 시뮬마다 **"물리적 정확성" vs "시각적 사실감"** 중 무게를 정한다.

## 6. 기술 스택

| 층 | 선택 | 역할 |
|---|---|---|
| 뼈대 | **Astro** | 구조·라우팅·정적 빌드 |
| 글 | **MDX** | 마크다운 글 안에 시뮬 컴포넌트 직접 삽입 |
| 인터랙티브 | **React** island | 시뮬을 컴포넌트로, 슬라이더·상태 관리 |
| 2D 시뮬 | **Canvas / SVG** | 광선·초점 등 광학 (물리엔진 불필요) |
| 3D 시뮬 | **Three.js** (react-three-fiber) | 3D 안구·투명 굴절 등, 필요할 때만 |
| 손그림 렌더 | **Rough.js** | 도형·시뮬을 손그림풍으로 (seed 고정해 안정화) |
| 수식 | **KaTeX** | 렌즈 방정식 등 |
| 스타일 | **Tailwind** | NAS choo-invest에 v4 전례, 재사용 |
| 배포 | 로컬 빌드 → **NAS 정적** | 기존 cloudflared 터널에 Public Hostname만 추가 |

- **접은 것**: 언리얼(웹 배포 사실상 불가). **블렌더는 폐기 아님** — 예쁜 3D 안구 모델(.glb)을 **제작하는 공방**으로 쓴다(렌더링 결과가 아니라 모델을 뽑아 Three.js에 넘김).
